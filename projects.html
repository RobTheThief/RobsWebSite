Restockr


This is web app designed to streamline ordering parts and help ensure the correct parts are ordered. It was made for Pair Mobile a mobile device repair company with 6 stores in Ireland and a warehouse. It uses the Repairshopr API to access Pairs database and create a unique list of parts used in a given period. It displays current stock levels for each store and the warehouse as well as an average stock level. A copy function was added to each product item.

nexTime Android App

nexTime Android App


nexTime is an android reminder app based on location and is built using React Native, with a custom geofencing method. As it uses backgroung location checking, permission is required from Google. The permissions request is still pending, however, A release APK is available from the Github repository but is only for demo purposes as the location service will not work yet.Release APK

Have you ever said to yourself: "The next time I am in town I will buy....", not knowing when you will be there next? Instead of having a reminder triggered by time, with this app you can be reminded the next time you are in a certain location. nexTime uses GPS, Wifi or Bluetooth to determine location or proximity to a chosen device. You can set reminders and add notes. For example create a reminder the nexTime you are in the city center and add notes with a shopping list.

Quotes

CRUDy Quotes API Project


To learn more about HTTP requests I decided to create my own API. This involved Node.js, Express to run the server and create the routes, and MongoDB to manage the JSON database. As my current hosting does not support VPS, Heroku was used to host the API.
The server renders HTML and can be accesed through the link below:
https://crudy-quotes.herokuapp.com/
Queries can also be sent through a third parties as CORS is allowed.
It uses HTTP/1.1 although it was initially intended to have HTTP/2 but an SSL certificate was required for Heroku.

ROBULATOR




Robulator!

After learning the basics of CSS, HTML and JavaScript I decided that my first project, should be a calculator. The only framework used was TailWindCSS.

It has 2 modes: Algebra and Robulator mode. In Algebra mode multiplication and division is done before addition and subtraction, which is the standard for most calculators.

Robulator mode essentially calculates from left to right. The idea was to reflect how you might calculate something in a story. E.g. I had 4 biscuits and then Rob gave me 2 more, but Tom stole 3. Harry then said that if I gave him one now he would give me back twice as many as I had left. I would then have 4 biscuits. 4+2-3-1*2 = 4. In Algebra mode the answer would be 1.
